# GBM A/B 显著性检验工作总结（阶段性）

本文档用于承接当前 `GBM A/B` 显著性检验功能的开发进展、性能现状与后续待办，方便后续继续迭代。

## 1. 当前目标与范围

- 目标：在 GBM 模拟资产世界中，对两组轮动策略参数（A/B）做配对统计检验。
- 主检验：
  - 配对置换检验（单侧，`H1: A > B`），指标为 `annualized_return(A-B)`。
  - Bootstrap 95% CI（`mean(diff)`）。
- A/B 参数要求：与主轮动参数体系保持一致（交互、语义、后端能力）。

## 2. 已完成内容

### 2.1 后端能力

- 已将 A/B 检验从简化轮动逻辑切换到主轮动引擎（`backtest_rotation`）路径，支持主轮动参数语义。
- `SimGbmAbStrategyParams` 已扩展支持以下能力字段：
  - `asset_momentum_floor_rules`
  - `asset_trend_rules`
  - `asset_bias_rules`
  - `asset_vol_index_rules`
  - `group_enforce` / `group_pick_policy` / `asset_groups` / `dynamic_universe`
- A/B 请求新增并行参数：
  - `n_jobs`（`0=自动`，其余为进程数）。
- A/B 世界数量下限已从 `50` 放宽到 `2`（用于快速回归与开发调试）。

### 2.2 前端与交互

- A/B 参数区已经升级为与主轮动同类结构：
  - 大标签：基础参数 / 入场控制 / 出场控制
  - 小标签：动量/趋势/乖离率（含入场与出场）+ 恐慌择时
- A/B 显著性面板新增：
  - `A/B 并行进程数(0=自动)` 输入项（`simAbJobs`）。
- A/B 参数持久化键已同步扩展，含新增参数项。

### 2.3 性能优化（已落地）

- 新增 `backtest_rotation(..., lightweight=True)` 轻量模式：
  - A/B 显著性仅保留必要输出（年化收益、平均暴露等）。
  - 跳过重型输出（rolling drawdown、period details、大量明细构建）。
- 同一世界中复用预计算：
  - 复用 OHLC 代理数据
  - 复用波动率指数代理数据
- 新增多进程并行评估世界：
  - 使用 `ProcessPoolExecutor`
  - 非 Windows 场景优先 `fork` 上下文（提升兼容性）
  - 并行不可用时自动回退单进程（保证功能可用性）

## 3. 性能诊断结论（实测）

## 3.1 热点来源（cProfile）

主要耗时不在 permutation/bootstrap，而在每个世界的策略回测重计算，尤其：

- `strategy/rotation.py::backtest_rotation`
- `baseline._rolling_max_drawdown`
- `baseline._max_drawdown_duration_days`
- pandas 大量索引切片赋值

### 3.2 关键基准（本机）

使用 10 年区间、`n_worlds=20`、`n_assets=4`、`n_perm=1000`、`n_boot=1000`：

- 优化后 + 单进程（`n_jobs=1`）：约 **55s**
- 优化后 + 双进程（`n_jobs=2`）：约 **38s**

同配置下 `n_worlds=2` 的回归测试（API/analysis）均在约 10 秒内完成。

## 5. 现状与限制

- 即使有轻量化与并行，`n_worlds` 与时间跨度仍近似线性增长。
- 对于 `3000 世界 x 50 年` 的目标，单次全量运行仍可能很慢（小时级），需进一步工程化策略。

## 6. 建议的下一步（优先级）

1. **分批累计执行（高优先）**
   - 将 3000 世界拆成多批（如 100~500/批）可续跑并增量合并统计。
   - 前端展示“已完成批次/累计样本”。

2. **预设档位（高优先）**
   - 快速 / 标准 / 严格 三档，自动填充 `n_worlds/n_perm/n_boot/n_jobs`。

3. **统计层进一步提速（中优先）**
   - 当样本足够大时，提供近似检验选项（或自适应降低 `n_perm/n_boot`）。

4. **并行策略增强（中优先）**
   - 根据机器核数给出推荐 `n_jobs`。
   - 增加“并行不可用原因”提示（当前为自动回退）。

5. **A/B UI 最后一轮一致化巡检（中优先）**
   - 逐项核对 A/B 与主轮动在文案、默认值、规则表行为的一致性。

## 8. 执行路线图（建议）

以下路线图按“先可用、后高性能、再工程化”的顺序推进。

### 阶段 A（1-2 天）：稳定性与一致性收口

目标：确保当前能力稳定可用，避免后续优化建立在不稳定基线之上。

- 任务 A1：A/B UI 与主轮动逐项对齐复核
  - 核对项：文案、默认值、规则表行为、开关禁用逻辑、N-of-M 提示。
  - 验收：A/B 任意一个参数项都能在主轮动区找到同语义映射。

- 任务 A2：参数快照与回放
  - 增加 A/B 参数快照导出（JSON）与导入（可选）。
  - 验收：同一参数快照重复运行结果可复现（同 seed）。

- 任务 A3：小样本回归固定化
  - 将 `n_worlds=2` 的回归测试保留为 CI 快速门禁。
  - 验收：两条 A/B 测试稳定在秒级通过。

### 阶段 B（2-4 天）：批处理与可续跑（优先级最高）

目标：把“大任务必须一次跑完”的模式改成“可分批、可中断、可续跑”。

- 任务 B1：后端分批执行接口
  - 新增批处理参数：`batch_size`、`batch_id`、`append_mode`。
  - 每批返回：新增样本数、累计样本数、当前统计摘要。
  - 验收：3000 世界可分 30 批执行，任意批失败后可重试续跑。

- 任务 B2：统计量增量合并
  - 把均值/方差/胜率/分布样本维护为可增量更新结构。
  - 验收：分批总结果与一次性全量结果在容差内一致。

- 任务 B3：前端批处理控制
  - 增加“开始/暂停/继续”与进度条、累计耗时展示。
  - 验收：页面刷新后可恢复任务状态（至少恢复参数和已完成批次信息）。

### 阶段 C（3-5 天）：性能提速与资源利用

目标：在相同统计精度下缩短总耗时。

- 任务 C1：并行策略自适应
  - `n_jobs=0` 自动按 CPU/内存估计推荐进程数。
  - 验收：默认设置下不显著拖垮机器且有明确提速。

- 任务 C2：世界级缓存（可选）
  - 对同一 `(start,end,n_assets,vol_range,seed,world_idx)` 的世界序列做缓存。
  - 验收：重复实验（只改 A/B 参数）明显提速。

- 任务 C3：统计计算优化
  - 提供近似模式（如较小 `n_perm/n_boot` 或分阶段加密）。
  - 验收：快速预筛模式与严格模式结论方向一致率可接受。

### 阶段 D（2-3 天）：实验工作流产品化

目标：让 A/B 检验形成可复用研究流程，而非一次性页面操作。

- 任务 D1：预设档位
  - 快速/标准/严格 三档（自动填充 `n_worlds/n_perm/n_boot/n_jobs`）。
  - 验收：用户一键切换档位并可自定义覆盖。

- 任务 D2：实验记录与复盘
  - 保存实验元数据（参数、时间、样本数、结果摘要、版本号）。
  - 验收：可查看历史实验列表并一键复跑。

- 任务 D3：结果可读性增强
  - 报告卡片加入“计算成本说明”（耗时、世界数、并行数、每世界平均耗时）。
  - 验收：用户可直接判断“结论可信度 vs 计算成本”。
