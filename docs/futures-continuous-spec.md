# 期货标的配置：真实合约拉取 + 主连合成与校验

需求规格说明（完整版）。真实合约与主连合成规则中，**88 / 888 / 889 及误差门禁**与 `src/etf_momentum/scripts/futures_continuous_replay.py` 对齐；**数据源**以产品约定为准（合约为腾讯财经；`*0` 比对基准需与 88 字段与频率可比）。

---

## 1. 背景与目标

1. 在**期货标的配置页**增加能力：在已获取**主力连续**价格数据的前提下，按规则拉取**真实可交易合约**的日度行情，用于在本地合成**简单主连（88）**、**前复权主连（888）**、**后复权主连（889）**。
2. 合成结果与真实合约数据**落在同一套期货价格数据库**中，通过**标的代码字母前缀 + 固定后缀**区分（见 §6）。
3. **真实合约**行情统一从**腾讯财经源**获取（字段见 §4）；**主连比对基准（\*0）**需与合成 88 在字段与频率上可对齐（实现上可与脚本一致：`*0` 主连序列；若合约与 \*0 均从腾讯拉取，则比对在腾讯语义下完成）。
4. 获取真实合约支持**串行 / 并行**；并行可配置**并发数，默认 5**。

---

## 2. 时间范围（相对已获取的主连）

在**已获取的主力连续数据**时间范围基础上：

| 维度     | 规则                                                                                                                                                                                       |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **起点** | 与已获取主连数据的**起点相同**                                                                                                                                                             |
| **终点** | 已获取主连数据的**终点**，再**向后扩展 12 个月**（与脚本中合约 `contract_end` 相对 `end_date` 推 12 个月的设计一致；实现需明确按**自然月**还是**交易日历**，并与现有主连区间截断方式一致） |

本窗口内需要覆盖「用于回放的**月份合约枚举**」及每个合约在窗口内的**日频行情**。

---

## 3. 真实可交易合约：全量 / 增量

### 3.1 全量

- **覆盖**本地已保存的、与本任务相关的合约数据（与「全量会覆盖所有本地已保存数据」一致）。
- 适用于首次灌库、纠错、用户主动全量刷新。

### 3.2 增量

- **按合约逐个**检查本地数据：
  - 若该合约本地数据已覆盖**满一个存续周期**（通常约**一年**；特殊品种如沥青等可为**两年**），则视为**已摘牌/无需再增量**。
  - 否则从本地**最后一个交易日**之后**继续增量**（与常见「从最后一根 K 的下一交易日」一致）。
- **品种特例**（一年 vs 两年）需**可配置或可维护**，不能写死全局一年。

---

## 4. 腾讯财经：真实合约字段与用途

从腾讯财经拉取的**每个真实合约、每个交易日**至少包含：

- **开、收、高、低**
- **结算价**
- **成交量、成交额、持仓量**

若某字段偶发缺失，需在实现上定义：**是否阻断合成 / 如何补缺**，并与 §5、§8 的比对字段一致。

## 5. 官方比对数据（基准主连 \*0）

与 `futures_continuous_replay.py` 对齐的规格：

| 项目         | 规则                                                                                                                                                               |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **代码**     | `main_symbol`，默认 **`{underlying}0`**（如 `LC` → `LC0`），可配置覆盖。                                                                                           |
| **含义**     | **未做本地回放合成的官方/行情商主连续序列**，用于与本地合成的 **88** 做误差分析。                                                                                  |
| **区间**     | 与 88 比对时，按 **`YYYYMMDD`** 定义的 **`[start_date, end_date]`** 与主连数据**做日期对齐**（脚本为截断到闭区间）。                                               |
| **对齐方式** | **88** 与 \*0 按 **`date` inner join**，仅**重叠交易日**参与误差统计。                                                                                             |
| **数据源**   | 需求要求合约走腾讯；*0 若也从腾讯拉取，则与 88 同源可比。脚本原型使用 `ak.futures_zh_daily_sina`（新浪），产品落地时以\*\*腾讯侧同语义 *0 品种代码与字段\*\*为准。 |

## 6. 存储命名（同一数据库）

| 后缀    | 含义                                       |
| ------- | ------------------------------------------ |
| **88**  | **简单主连**（无复权），由真实合约回放合成 |
| **888** | **前复权**主连                             |
| **889** | **后复权**主连                             |

- **真实合约**、**88 / 888 / 889** 均存**同一期货价格库**，用**标的根代码 + 后缀**区分。
- **简单主连与前后复权主连的合成**均为**全量重算**（每次跑完整条序列，不做「只补最后几天」的增量合成，与脚本行为一致）。**落库时**先按 `code+adjust` 删除该品种已保存的 `88` / `888` / `889` 行，再写入本次整段结果，使库中主连与当前一次合成结果一致（无历史尾段残留）。

## 7. 简单主连（88）合成规则（与脚本一致）

**输入**：各交割月合约在窗口内的日频数据；**主力判定依赖持仓量 `hold`**。

1. **持仓横截面**：所有合约按日形成 `date × contract` 的持仓矩阵（多合约同日取 last 聚合）。
2. **首日**：当日**持仓最大**的合约为当日主力；若无有效持仓则该日可能无输出（与脚本边界一致）。
3. **次日起**：设上一日主力为 `cur`，在**上一交易日**的持仓截面上：
   - `eligible_best` = 在「允许成为主力」的合约集合中持仓最大者；允许集合为：`cur` 本身（可延续）以及**历史上从未当过主力**的合约（见 `docs/futures-continuous-rules.md`：一个合约至多一次成为主力）。
   - 若 `eligible_best ≠ cur` 且 `eligible_best_hold > switch_threshold × cur_hold`，则**换月**为 `eligible_best`，否则**保持** `cur`。
4. **兜底**：若 `cur` 无效或当日该合约无行情，则在**当日**持仓截面上按与上条相同的「允许成为主力」规则取持仓最大者；若该集合为空则退回无过滤的持仓最大（极端情形）。
5. **输出**：每个交易日一行，取当日主力合约当日的 **`open, high, low, close, volume, hold, settle`**（及业务需要的其它列）；并生成**换月表**：相邻两日主力代码变化则记录 `date, from_symbol, to_symbol`。

**参数**：`switch_threshold`，脚本默认 **1.1**（可配置）。

## 8. 误差比对与「通过」条件（88 vs \*0）

与脚本 `_calc_error_stats` + `_evaluate_usability` 一致。

### 8.1 统计字段

对以下字段在 inner join 样本上计算（双方均非空）：

`open, high, low, close, volume, amount, hold, settle`

对每个字段：`diff = 88 - main0`，并计算 **MAE、RMSE、max_abs、MAPE、P95 APE**（APE = `|diff|/|main0|`，`main0=0` 按缺失处理）。

### 8.2 门禁（是否允许进入 888/889）

- **关键字段**（参与门槛）：`open, high, low, close, settle`。
- 对每个关键字段，若 `n > 0`，需同时满足：
  - **MAPE ≤ `usable_rel_mean_max`**，默认 **0.005（0.5%）**
  - **P95 APE ≤ `usable_rel_p95_max`**，默认 **0.02（2%）**
- **全局**：在 5 个关键字段中，**至少 `usable_min_fields` 个**（默认 **4**）有有效样本且**未**触发上述超标；且**不能**有关键字段已覆盖却超阈值。
- 若 \*0 或 88 为空、或 inner join 无重叠 → **比对不可用**，**不允许**进入 888/889 落库（`usable = false`）。

报告层可额外输出收盘价差分摘要等，**以门禁逻辑为准**。

## 9. 前复权主连（888）合成规则（与脚本一致）

在 **88**、**换月表**、**分合约行情 panel** 上：

1. 对每个换月事件（换月日为 `d`，从 `from` 到 `to`）：取换月**前一交易日** `prev_d`。
2. **`pre_delta = close(to, prev_d) - close(from, prev_d)`**（新主力 − 旧主力，使 T−1 及以前整体平移后与换月后合约尺度衔接）；任一侧缺失则该次不累计。
3. **累计调整量** `pre_adj`：每个 `pre_delta` 加到**所有日期 ≤ `prev_d`** 上（历史向前累加）。
4. **价格**：对 **`open, high, low, close, settle`**，新值 = 原值 + `pre_adj(date)`。
5. **成交额 `amount`**：**不复权，置为 0**（与脚本注释一致）。

---

## 10. 后复权主连（889）合成规则（与脚本一致）

1. 对每个换月日 `d`：**`post_delta = open(from, d) - open(to, d)`**。
2. **累计调整量** `post_adj`：每个 `post_delta` 加到**所有日期 ≥ `d`**（从换月起向后累加）。
3. **价格**：同样对 **`open, high, low, close, settle`** 加上 `post_adj`；**`amount = 0`**。

---

## 11. 流程顺序（业务）

1. 已具备主连时间窗 → 按 §2 确定合约窗口 → 按 §3、§4 拉取/更新**真实合约**（腾讯）。
2. **全量**合成 **88**（及换月表）。
3. 拉取或读取 **\*0** 基准主连（§5），与 **88** 做 §8 误差分析。
4. **仅当 §8 通过**，再**全量**合成并落库 **888、889**（§9–§10）。
5. 三类主连与合约数据**同库、后缀区分**（§6）。

---

## 12. 性能与执行模式

- 真实合约获取：**串行**或**并行**。
- **并行**：可配置**最大并发数**，**默认 5**。
- 合成任务：88/888/889 为 CPU/内存型全量计算，是否与拉取并行、资源上限，可在实现阶段定义。
